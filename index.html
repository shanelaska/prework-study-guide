HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML is the foundation of every web page. It provides the structure and meaning for the content you see on websites.
  • HTML uses elements, which are like building blocks, to create different parts of a webpage. For example, a paragraph is created using the <p> element.
  • Tags are used to define elements. They come in pairs: an opening tag (e.g., `<p>`) and a closing tag (e.g., `</p>`).
  • Headings: `<h1>`, `<h2>`, `<h3>`, etc. for titles and subtitles.
  • Paragraphs: `<p>` for blocks of text.
  • Links: `<a>` to create hyperlinks to other pages or resources.
  • Images: `<img>` to display pictures.
  • Lists: `<ul>` for unordered lists (bullets), `<ol>` for ordered lists (numbers).
  • Forms: `<form>`, `<input>`, `<button>` for user input.

CSS

  • CSS makes your web pages look good! It controls the visual presentation of your HTML content.
  • CSS uses rules to define styles for specific elements. A rule consists of a selector (to identify which elements to style) and declarations (the specific styles to apply).
  • Common CSS properties include:
  • `color`: sets the text color.
  • `font-size`: sets the size of the text.
  • `background-color`: sets the background color of an element.
  • `width` and `height`: control the dimensions of an element.
  • `margin`: adds space around an element.
  • `padding`: adds space inside an element.
  • Inline: Directly in the HTML tag (e.g., `<p style="color: blue;">`).
  • Internal: Within a `<style>` tag in the HTML's `<head>` section.
  • External: In a separate `.css` file linked to your HTML using the `<link>` tag.

Git

  • Git is a version control system that lets you track changes to your code over time.
  • This makes it easy to collaborate with others, revert to previous versions, and see a history of your project.
  • Key Git commands include:
  • `git init`: Initializes a new Git repository.
  • `git add`: Adds changes to the staging area.
  • `git commit`: Creates a new commit with the staged changes.
  • `git push`: Uploads local commits to a remote repository (e.g., GitHub).
  • `git pull`: Downloads remote changes to your local repository.
  • `git branch`: Creates, lists, and deletes branches.

JavaScript

  • JavaScript is a programming language that brings interactivity and dynamic behavior to web pages.
  • It can do things like:
    • Change the content of a page after it has loaded.
    • Respond to user actions (like clicks, hovers, form submissions).
    • Validate user input in forms.
    • Create animations and visual effects.
    • Send and receive data from servers (AJAX).
  • JavaScript code can be embedded directly in your HTML using the `<script>` tag or written in separate `.js` files and linked using the `<script>` tag.
